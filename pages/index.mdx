        ### ViewNext
        En Junio de 2018, comencé de mi partida como desarrollador de manera profesional en el puesto de programador junior. El proyecto para el que trabajé era el mismo en el que realicé mis prácticas, un proyecto de LaCaixa, un aplicativo web en el patrón modelo-vista-controlador usando el framework Spring de Java.
        En el hacia desde la consulta SQL a la base de datos usando jdbc y MyBatis usando formato XML. Después DTO's, POJOS, servicios, servicios impl, Controladores, JSP's y el patrón de inyección de dependencias en el backend. El IDE usado era Spring Tools de Eclipse, utilizabamos jetty como servidor HTTP y así poder llegar a la capa frontal donde
        la mágia se hacia con html, css con el preprocesador sass y javascript con librearías como JQuery y DataTables. Para gestión de errores y trazas usabamos Kibana, el control de versiones era con SubVersión, la integración continua con Jenkins y la metodologia de trabajo era en cascada.

        ### Plexus
        En febrero de 2021 tomé la decisión de buscar un trabajo 100% remoto que me permitiese estar en mi ciudad natal y poder hacer obras de mejora de la vivienda familiar y habilitar una de sus plantas. Fue cuando recibí una oferta de teletrabajo 100% y con el stack tecnologico que buscaba. Para entrar tuve que hacer una prueba de código que soliciaba el cliente (OpenBank),
        actualmente este es mi proyecto donde se trabaja con metodologia agile(Skrum y tools de Atlassian), yo formo parte del equipo de Front donde hacemos evolutivos y mantenimiento de la parte de inversiones que esta totalmente hecha con React usando una arquitectura de BFF y Microfrontales.
        Usamos storybook para documentar nuestros componentes, jest con enzyme y react-testing-library para hacer testing unitario y de integración. Para el consumo de información utilizamos fetch y react-query y para la validación de los contratos Yup/Swagger. El proyecto cuenta con eslint, prettier y husky para el control de la guia de estilos común y utiliza Flow para el típado estático.
        Para la lógica de negocio y el contexto de la aplicación se utiliza ContextAPI por flujos y para la parte de la sesión de los usuarios, Redux. El proyecto utiliza feature flags y utiliza google analitycs. Los textos y traducciones se hacen a través de Poeditor.
